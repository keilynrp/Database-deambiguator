# Sprint Daily Update Template

Plantilla diaria para coordinacion entre agentes durante un sprint.

## Uso

- una entrada por agente por dia
- escribir corto y verificable
- registrar bloqueos y cambios de alcance el mismo dia
- si una historia cambia de estado, actualizar tambien la historia y el sprint

## Template

```md
# Daily Update - YYYY-MM-DD

## Agent

- Nombre o perfil: `Agent X`
- Sprint: `SPRINT-XXX`

## 1. Historias activas

- `US-XXX` - Estado actual
- `US-YYY` - Estado actual

## 2. Avance de hoy

- que se completo
- que se avanzo parcialmente
- que evidencia quedo

## 3. Bloqueos

- bloqueo actual
- impacto
- dependencia o decision requerida

## 4. Riesgos detectados

- riesgo
- impacto probable
- mitigacion propuesta

## 5. Plan inmediato

- siguiente paso 1
- siguiente paso 2

## 6. Archivos o areas tocadas

- `path/o/modulo`
- `path/o/modulo`

## 7. Validacion realizada

- test ejecutado
- validacion manual
- pendiente de validar

## 8. Necesita coordinacion con

- `Agent X` por motivo
- `Agent Y` por motivo

## 9. Decision o cambio de alcance

- decision tomada
- alcance agregado o removido
- justificacion
```

## Regla de calidad

Una buena actualizacion diaria:

- permite entender progreso en 30 segundos
- deja claro si alguien esta bloqueado
- muestra evidencia minima
- evita textos narrativos largos
